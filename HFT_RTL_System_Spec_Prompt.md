# SYSTEM SPECIFICATION PROMPT
## Target: Sub-Microsecond HFT Trading Engine — RTL Implementation
### Instruction Document for Autonomous Coding Agent

---

## PREAMBLE — READ BEFORE GENERATING A SINGLE LINE

This is not a general networking design. You are building a **Tick-to-Trade** engine. Every architectural decision — pipeline depth, interface choice, module boundary, register placement — must be evaluated through a single lens: **deterministic, minimum-latency path from raw wire bits to outbound order packet.**

Ambiguity is not acceptable. If the spec does not specify, assume the lower-latency option.

---

## 1. ARCHITECTURE PARADIGM: HEADLESS, CUT-THROUGH PIPELINE

**Mandate:** The design must implement a **headless, cut-through** forwarding architecture.

**What this means, precisely:**

- **No frame buffering.** Parsing, risk evaluation, and output formatting must begin before the frame tail is received. The pipeline must accept and process frame bytes as they arrive from the MAC, one clock cycle at a time.
- **No CPU in the datapath.** There is no OS, no DMA controller, no SoC bus, no memory-mapped register file between wire and decision. Configuration is loaded at reset. Runtime decision logic is purely combinational and registered — no firmware calls.
- **No AXI, AHB, APB, or any bus fabric in the critical path.** These add arbitration latency. They are prohibited in the datapath. Configuration registers may use a separate, non-timing-critical path (e.g., a simple shift register chain loaded at reset).
- **Pipelined, not iterative.** The datapath is a strictly ordered sequence of pipeline stages. Each stage has exactly one clock cycle of latency. There are no feedback loops, no retry paths, no state machines that stall waiting for downstream readiness — with the single exception of the Risk Gate's hold-off mechanism, which must assert upstream backpressure within the same clock cycle that a violation is detected.
- **Cut-through forwarding:** The Packet Formatter must begin emitting the outbound frame header **no later than 4 cycles after the inbound SFD (Start of Frame Delimiter) is detected**, regardless of whether the full inbound frame has been received. The trailing FCS is computed incrementally.

## 2. INTERFACE DEFINITIONS — RAW BIT-LEVEL 10GbE MAC/PCS

Do **not** wrap these in AXI-Stream or any standard bus protocol. Every bus handshake signal that isn't strictly necessary is a cycle you cannot afford.

### 2.1 Inbound Interface (from 10GbE PCS to Top-Level)

```systemverilog
// Clock: 156.25 MHz (64b/66b PCS clock domain)
// All signals synchronous to rising edge of clk_pcs

input  logic        clk_pcs,           // 156.25 MHz PCS recovered clock
input  logic        rst_n,             // Active-low synchronous reset

// Raw 64-bit PCS word interface (post-64b/66b decode, pre-MAC framing)
input  logic [63:0] pcs_rxdata,        // 64 bits per cycle, LSB = earliest bit
input  logic [7:0]  pcs_rxctl,         // Per-byte control: 1 = control char, 0 = data
input  logic        pcs_rx_valid,      // Word on bus is valid this cycle
input  logic        pcs_block_lock,    // PCS has achieved 64b/66b block lock

// Derived MAC signals — generated by thin MAC shim (see §2.3)
input  logic        mac_sof,           // Start of Ethernet frame (SFD detected)
input  logic        mac_eof,           // End of Ethernet frame (EOF /T/ char detected)
input  logic [2:0]  mac_eof_bytecount, // Valid bytes in final word [0=8, 1..7=N]
input  logic        mac_fcs_valid,     // Asserted with mac_eof if CRC passed
```

### 2.2 Outbound Interface (from Top-Level to 10GbE PCS)

```systemverilog
output logic [63:0] pcs_txdata,        // 64 bits per cycle to PCS encoder
output logic [7:0]  pcs_txctl,         // Per-byte control indicators
output logic        pcs_tx_valid,      // Valid word on output bus
output logic        pcs_tx_sof,        // Start of outbound frame
output logic        pcs_tx_eof,        // End of outbound frame
output logic [2:0]  pcs_tx_eof_bytes,  // Valid bytes in final word
```

### 2.3 Thin MAC Shim — Specification

Implement a single module `mac_shim` that:
- Detects the Ethernet Preamble (`0x55_55_55_55_55_55_55_D5`) and asserts `mac_sof` on the cycle the SFD byte (`0xD5`) appears in the 64-bit word.
- Detects the PCS `/T/` (Terminate) control character and asserts `mac_eof`.
- Computes CRC-32 incrementally as bytes arrive; asserts `mac_fcs_valid` coincident with `mac_eof` if the received FCS matches.
- Adds **zero cycles of buffering** — it is a pure combinational decoder with one register stage for output alignment.
- Does **not** strip the preamble from the datastream — that is the Header Stripper's job.

## 3. MODULE DECOMPOSITION

The top-level module `hft_engine` instantiates exactly the following sub-modules in pipeline order. No additional hierarchy is permitted in the datapath.

```
[PCS] → mac_shim → hdr_stripper → field_aligner → sym_id_mapper → risk_gate → pkt_formatter → [PCS]
```

Each module receives the same `clk_pcs` and `rst_n`. No module introduces its own clock domain. No CDC logic exists in the datapath.

---

### 3.1 Module: `hdr_stripper`

**Function:** Remove Ethernet preamble, DA, SA, EtherType, and IP/UDP headers. Emit only the UDP payload (market data message body) to downstream stages.

**Interface:**

```systemverilog
module hdr_stripper (
    input  logic        clk_pcs,
    input  logic        rst_n,

    // From mac_shim
    input  logic [63:0] rx_data,
    input  logic        rx_valid,
    input  logic        rx_sof,
    input  logic        rx_eof,
    input  logic [2:0]  rx_eof_bytes,

    // To field_aligner
    output logic [63:0] payload_data,
    output logic        payload_valid,
    output logic        payload_sof,    // First word of UDP payload
    output logic        payload_eof,
    output logic [2:0]  payload_eof_bytes,
    output logic        frame_err       // Asserted if EtherType != 0x0800, 
                                        // IP proto != UDP, or bad length
);
```

**Implementation constraints:**
- Header byte offsets are **compile-time fixed constants**. Ethernet+IPv4+UDP header = 42 bytes = 5.25 × 64-bit words. Handle the fractional word boundary using a registered barrel shift, not a loop.
- The preamble occupies the first 64-bit word following `rx_sof`. Strip it with a constant-offset suppress, not a search.
- IP header length field: **assume fixed 20-byte IP header.** Variable-length IP options are not supported and must be flagged as `frame_err`.
- `frame_err` must be asserted in the **same cycle** as the offending byte is processed. It must propagate to kill the downstream pipeline without requiring a frame flush handshake.
- **Cycle budget: 2 cycles from `rx_sof` to first `payload_valid`.**

---

### 3.2 Module: `field_aligner`

**Function:** Extract specific fixed-offset fields from the UDP payload and present them as named, word-aligned signals to downstream logic. This is the stage that converts a byte stream into a typed struct.

**Interface:**

```systemverilog
module field_aligner (
    input  logic        clk_pcs,
    input  logic        rst_n,

    input  logic [63:0] payload_data,
    input  logic        payload_valid,
    input  logic        payload_sof,
    input  logic        payload_eof,
    input  logic        frame_err,

    // Extracted fields — registered, valid when field_valid asserted
    output logic [15:0] msg_type,       // Bytes 0–1 of UDP payload
    output logic [63:0] instrument_id,  // Bytes 2–9
    output logic [63:0] price,          // Bytes 10–17 (fixed-point, big-endian)
    output logic [31:0] quantity,       // Bytes 18–21
    output logic [7:0]  side,           // Byte 22 (0x42='B', 0x53='S')
    output logic        field_valid,    // All fields above are valid this cycle
    output logic        field_err       // Propagated frame_err or alignment failure
);
```

**Implementation constraints:**
- All field offsets are **static parameters** passed at instantiation. No runtime offset calculation.
- Fields that span a 64-bit word boundary must be assembled using two registered partial captures that are concatenated in the following cycle. Maximum 1-cycle cost for any cross-boundary field.
- Endian swapping (big-endian wire to little-endian internal) must be done here, once, using a static byte-reversal (`{data[7:0], data[15:8], ...}`). No byte-swap logic may appear downstream.
- **Cycle budget: 1 cycle from first `payload_sof` to `field_valid`** (for fields fully contained in the first payload word). Cross-boundary fields: **2 cycles maximum.**

---

### 3.3 Module: `sym_id_mapper`

**Function:** Translate the raw `instrument_id` field (exchange-native instrument token) to an internal symbol index used throughout the risk and order management logic.

**Interface:**

```systemverilog
module sym_id_mapper #(
    parameter int SYMBOL_TABLE_DEPTH = 1024,  // Must be power of 2
    parameter int SYMBOL_ID_WIDTH    = 10     // log2(SYMBOL_TABLE_DEPTH)
) (
    input  logic        clk_pcs,
    input  logic        rst_n,

    input  logic [63:0] instrument_id,
    input  logic        field_valid,
    input  logic        field_err,

    output logic [SYMBOL_ID_WIDTH-1:0] symbol_idx,
    output logic        sym_valid,
    output logic        sym_miss,       // instrument_id not in table → kill frame
    output logic        sym_err         // Propagated field_err
);
```

**Implementation constraints:**
- Implement as a **direct-mapped lookup table** instantiated as a synchronous single-port SRAM or synthesis-inferred `logic` array. Do not use CAM (Content Addressable Memory) or hash chaining — both have variable latency.
- The table is indexed by the lower `SYMBOL_ID_WIDTH` bits of `instrument_id`. If a collision or miss occurs (tag mismatch), assert `sym_miss` and propagate a kill signal downstream.
- Table contents are loaded at reset via a dedicated serial load interface (not part of the datapath critical path). The table is read-only during operation.
- **Cycle budget: 1 cycle** (registered SRAM read).

---

### 3.4 Module: `risk_gate`

**Function:** Evaluate inbound order fields against pre-loaded risk limits. This is the only module permitted to assert pipeline hold-off. It must do so in the same cycle a violation is detected — no lookahead, no post-hoc kill.

**Interface:**

```systemverilog
module risk_gate #(
    parameter int SYMBOL_TABLE_DEPTH = 1024,
    parameter int SYMBOL_ID_WIDTH    = 10,
    parameter int PRICE_WIDTH        = 64,
    parameter int QTY_WIDTH          = 32
) (
    input  logic        clk_pcs,
    input  logic        rst_n,

    input  logic [SYMBOL_ID_WIDTH-1:0] symbol_idx,
    input  logic [PRICE_WIDTH-1:0]     price,
    input  logic [QTY_WIDTH-1:0]       quantity,
    input  logic [7:0]                 side,
    input  logic                       sym_valid,
    input  logic                       sym_miss,
    input  logic                       sym_err,

    output logic        risk_pass,      // Frame cleared — propagate to formatter
    output logic        risk_kill,      // Frame blocked — assert same cycle as violation
    output logic [3:0]  kill_reason,    // Encoded violation type
    output logic        risk_err        // Propagated upstream errors
);
```

**Risk checks to implement (all must complete within 1 cycle, fully parallel):**

| Check | Condition | kill_reason |
|-------|-----------|-------------|
| Price floor | price < risk_price_floor[symbol_idx] | 4'h1 |
| Price ceiling | price > risk_price_ceil[symbol_idx] | 4'h2 |
| Quantity limit | quantity > risk_qty_max[symbol_idx] | 4'h3 |
| Kill switch | risk_global_kill == 1 | 4'h4 |
| Symbol miss | sym_miss == 1 | 4'h5 |
| Upstream error | sym_err == 1 | 4'hF |

**Implementation constraints:**
- All six checks are evaluated **in parallel in combinational logic**, registered at the output. No priority encoder chain.
- Risk limit tables are synthesis-inferred arrays, pre-loaded at reset. They are not writable during operation.
- `risk_kill` must be a registered output that is valid **1 cycle after `sym_valid`** — this is the kill propagation latency budget.
- The global kill switch (`risk_global_kill`) is a single registered flip-flop driven by an off-path input. It has no timing relationship to the datapath clock but must be captured synchronously.
- **Cycle budget: 1 cycle from `sym_valid` to `risk_pass`/`risk_kill`.**

---

### 3.5 Module: `pkt_formatter`

**Function:** Construct the outbound order packet and begin transmission as soon as the risk gate passes. The output must be emitted before the inbound frame has finished arriving.

**Interface:**

```systemverilog
module pkt_formatter #(
    parameter int SYMBOL_ID_WIDTH = 10,
    parameter int PRICE_WIDTH     = 64,
    parameter int QTY_WIDTH       = 32
) (
    input  logic        clk_pcs,
    input  logic        rst_n,

    input  logic [SYMBOL_ID_WIDTH-1:0] symbol_idx,
    input  logic [PRICE_WIDTH-1:0]     price,
    input  logic [QTY_WIDTH-1:0]       quantity,
    input  logic [7:0]                 side,
    input  logic                       risk_pass,
    input  logic                       risk_kill,

    // To PCS TX
    output logic [63:0] pcs_txdata,
    output logic [7:0]  pcs_txctl,
    output logic        pcs_tx_valid,
    output logic        pcs_tx_sof,
    output logic        pcs_tx_eof,
    output logic [2:0]  pcs_tx_eof_bytes
);
```

**Implementation constraints:**
- The outbound Ethernet+IP+UDP headers are **static templates** stored in a parameter array. Only the payload fields (price, quantity, side, symbol) are substituted per-frame.
- IP and UDP checksums: IP header checksum is pre-computed for the static header and updated only for the TTL and length fields (both static). UDP checksum is disabled (set to 0x0000 per RFC 768).
- Ethernet FCS is computed incrementally as words are emitted, appended at `pcs_tx_eof`.
- Output must begin emitting (`pcs_tx_sof`) **no later than 1 cycle after `risk_pass`.**
- If `risk_kill` is asserted, the formatter must suppress all output and reset its internal state within 1 cycle. No partial frames may appear on the TX bus.
- **Cycle budget: 1 cycle from `risk_pass` to first `pcs_tx_sof`.**

## 4. TIMING CONSTRAINTS AND CYCLE BUDGET SUMMARY

All budgets are measured from the **rising edge of `clk_pcs`** at which the trigger event is registered.

| Stage | Trigger | Output | Max Latency |
|-------|---------|--------|-------------|
| mac_shim | PCS block lock + preamble detect | `mac_sof` | 1 cycle |
| hdr_stripper | `mac_sof` | First `payload_valid` | 2 cycles |
| field_aligner | `payload_sof` | `field_valid` (in-word fields) | 1 cycle |
| field_aligner | `payload_sof` | `field_valid` (cross-word fields) | 2 cycles |
| sym_id_mapper | `field_valid` | `sym_valid` | 1 cycle |
| risk_gate | `sym_valid` | `risk_pass` / `risk_kill` | 1 cycle |
| pkt_formatter | `risk_pass` | `pcs_tx_sof` | 1 cycle |

**Total deterministic pipeline depth (best case, in-word fields): 7 cycles = 44.8 ns at 156.25 MHz.**

This is the number that must not grow. Any architectural change that adds a cycle to this path must be explicitly justified and approved before implementation.

**Parser completion constraint:** The pipeline must assert `field_valid` within **3 cycles of `mac_sof`** regardless of field layout. This is a hard architectural constraint, not a target.

## 5. CODING STANDARDS

### 5.1 Language

- **SystemVerilog only.** No VHDL, no mixed-language instantiation.
- All synthesizable RTL must be written as behavioral SystemVerilog targeting synthesis. No `$display`, `$monitor`, or `$random` in synthesizable code.
- Testbench and assertion code may use the full SystemVerilog feature set including `program` blocks, `clocking` blocks, and SVA.

### 5.2 Clock and Reset

- **One clock domain: `clk_pcs` at 156.25 MHz.** No gated clocks, no derived clocks, no `posedge/negedge` mixing.
- **One reset: `rst_n`, active-low, synchronous to `clk_pcs`.** No asynchronous reset anywhere in the datapath.
- Every flip-flop must have an explicit synchronous reset value. Uninitialized state is not acceptable.

### 5.3 Forbidden Constructs

The following are explicitly prohibited in any synthesizable module:

| Construct | Reason |
|-----------|--------|
| `for` / `while` / `repeat` loops with variable iteration count | Non-deterministic latency |
| `case` statements with a default that implies variable handling | May infer priority encoders with non-constant depth |
| `#delay` in synthesizable code | Not synthesizable |
| Latches (any `always` block without complete sensitivity or missing `else`) | Non-deterministic hold |
| `if/else` chains deeper than 3 levels | Infers mux chains; flatten to parallel logic |
| Blocking assignments in `always_ff` blocks | Race conditions |
| Non-blocking assignments in `always_comb` blocks | Simulation/synthesis mismatch |
| RAM inference with read-during-write behavior | Undefined latency |
| `$cast`, dynamic arrays, queues, classes in synthesizable code | Not synthesizable |

### 5.4 Naming Conventions

- Module names: `snake_case`, descriptive, no abbreviations except those defined in this document.
- Signal names: `snake_case`. Registered signals must carry the suffix `_r`. Combinational: no suffix.
- Parameters: `UPPER_SNAKE_CASE`.
- All ports must be typed (`logic`, not `wire` or `reg`).
- All port directions must be explicit (`input logic`, `output logic`, `inout` is prohibited).

### 5.5 File Structure

- One module per file. File name matches module name exactly.
- Each file begins with a standard header comment block:
  ```systemverilog
  // Module     : <module_name>
  // Description: <one-line function description>
  // Latency    : <N cycles>
  // Clock      : clk_pcs @ 156.25 MHz
  // Reset      : rst_n, active-low synchronous
  ```
- All parameters must be documented inline with a comment explaining their legal range and the synthesis implication of changing them.

### 5.6 Assertions

Every module must include a minimum set of concurrent SVA assertions bound in a `module_name_assertions.sv` bind file:

- **Valid stability:** Once `*_valid` is asserted, it must not deassert until `*_eof` is seen.
- **No-gap rule:** There must be no gap cycles in `payload_valid` between `payload_sof` and `payload_eof`.
- **Kill propagation:** If `risk_kill` asserts, `pcs_tx_valid` must deassert within 1 cycle.
- **Cycle budget:** Each module's output must appear within its specified latency budget, expressed as a bounded `##N` sequence in SVA.
